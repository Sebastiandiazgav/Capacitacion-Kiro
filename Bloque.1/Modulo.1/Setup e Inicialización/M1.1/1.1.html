01 — Definición

¿Qué es Kiro?

Kiro es un IDE agéntico lanzado por Amazon Web Services en julio de 2025. Está construido sobre VS Code OSS y potenciado por los modelos Claude de Anthropic. Su propósito central es cerrar la brecha entre el "vibe coding" (prototipos rápidos con IA) y el software de producción real, aplicando rigor de ingeniería a cada paso del desarrollo.

🧠

Agente con contexto completo

Kiro entiende tu proyecto completo: archivos, dependencias, historial y convenciones. No solo completa líneas, razona sobre el sistema.

📋

Desarrollo guiado por especificaciones

Transforma lenguaje natural en requisitos, diseño y lista de tareas antes de escribir una sola línea de código.

🔗

Ecosistema AWS nativo

Integración directa con Lambda, DynamoDB, CloudWatch, SageMaker y más. Infraestructura como código con CDK, SAM o Terraform.

🔒

Privacidad primero

Arquitectura de seguridad que no entrena modelos con tu código. Tu propiedad intelectual permanece tuya.

02 — El Problema

El desarrollo con IA tiene fricciones reales

Las herramientas de IA actuales aceleran la escritura de código, pero crean nuevos problemas que frenan a los equipos de ingeniería.

🌀

Código sin documentación

El código generado por IA rara vez incluye contexto de decisiones de diseño, creando deuda técnica invisible desde el primer commit.

🎯

Prompts imprecisos = resultados impredecibles

Sin una especificación clara, el agente interpreta la intención del desarrollador de forma diferente en cada sesión.

🔄

Iteraciones infinitas

Sin un plan estructurado, los equipos pasan más tiempo corrigiendo el output de la IA que construyendo funcionalidades reales.

🏗️

Prototipos que no escalan

El "vibe coding" produce demos funcionales pero frágiles. Llevarlos a producción requiere reescrituras completas.

03 — Características

Todo lo que necesitas para construir en serio

Kiro combina la velocidad del desarrollo asistido por IA con las prácticas de ingeniería que los equipos profesionales necesitan.

📝

Modo Spec (Especificaciones)

Define tu intención en lenguaje natural. Kiro genera automáticamente requisitos, documento de diseño y lista de tareas antes de escribir código.

💬

Modo Vibe (Chat agéntico)

Conversación fluida con el agente para exploración, preguntas sobre el código y generación rápida sin estructura formal.

🤖

Autopilot & Supervised

Autopilot ejecuta tareas end-to-end de forma autónoma. Supervised presenta cada cambio como hunk individual para aprobación granular.

⚡

Hooks agénticos

Automatizaciones event-driven: al guardar un archivo, crear uno nuevo o completar una tarea, el agente ejecuta acciones en segundo plano.

🧭

Steering (Directrices)

Archivos de contexto persistente que inyectan estándares del equipo, convenciones de código y guías de arquitectura en cada interacción.

🔌

MCP (Model Context Protocol)

Conecta el agente con APIs externas, bases de datos y servicios de forma segura. Extiende las capacidades de Kiro sin límites.

☁️

Integración AWS profunda

Genera código para Lambda, diseña esquemas DynamoDB, analiza logs de CloudWatch y conecta con SageMaker sin salir del IDE.

📚

Documentación en tiempo real

Mantiene la documentación sincronizada con el código automáticamente. Cada cambio queda registrado con su contexto de decisión.

04 — Comparación

Kiro vs Claude Code

Ambas herramientas usan modelos Claude de Anthropic, pero su filosofía de trabajo es radicalmente diferente. La elección depende del perfil del equipo y el tipo de proyecto.

Criterio ⬡ Kiro Claude Code
Entorno de trabajo IDE visual (VS Code OSS) Terminal / CLI
Curva de aprendizaje ✓ Baja — interfaz familiar ~ Requiere dominio de terminal
Desarrollo guiado por specs ✓ Nativo y estructurado ✗ No incluido
Automatizaciones (Hooks) ✓ Event-driven integrado ✗ Manual / scripts externos
Directrices de equipo (Steering) ✓ Archivos de contexto persistente ~ CLAUDE.md básico
Integración AWS ✓ Nativa y profunda ~ Vía MCP / manual
Control granular de cambios ✓ Modo Supervised (hunk a hunk) ✓ Plan Mode
Soporte multi-modelo ✓ Claude Sonnet 3.7 / 4.0 ✓ Claude Opus / Sonnet
Ideal para Equipos con flujos estructurados Devs avanzados en terminal
Documentación automática ✓ Sincronizada en tiempo real ✗ No incluida
