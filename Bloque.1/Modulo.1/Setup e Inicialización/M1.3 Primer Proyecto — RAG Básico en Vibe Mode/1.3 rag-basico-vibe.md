# 1.3 Workshop: RAG Básico en Vibe Mode

## Objetivo

Construir un RAG funcional que responda consultas sobre documentación interna de AFP UNO, usando Kiro en modo Vibe con LangChain + Bedrock.

**Tiempo:** 30 minutos  
**Modo:** Vibe (chat agéntico)  
**Stack:** Node.js + TypeScript + LangChain + Bedrock (Claude + Titan Embeddings)

---

## Prerrequisitos

- ✅ Kiro instalado (M1.1)
- ✅ Credenciales AWS del sandbox configuradas (`aws configure` o variables de entorno)
- ✅ Node.js 18+ instalado
- ✅ Acceso a Amazon Bedrock habilitado en la cuenta sandbox (modelos: Claude Sonnet, Titan Embed)

---

## Paso 1: Crear workspace

1. Crear una carpeta nueva para el proyecto:
```bash
mkdir rag-afp-uno
cd rag-afp-uno
```

2. Abrir en Kiro: **File → Open Folder → seleccionar `rag-afp-uno`**

## Paso 2: Prompt al agente (Vibe Mode)

### 2.1 — Verificar dependencias instaladas

Antes de comenzar, asegúrate de tener instalado lo siguiente:

**Software requerido:**

| Herramienta | Para qué | Verificar con |
|-------------|----------|---------------|
| Node.js 18+ | Runtime del proyecto | `node --version` |
| npm | Gestor de paquetes (viene con Node) | `npm --version` |
| AWS CLI | Configurar credenciales para Bedrock | `aws --version` |
| Git | Control de versiones | `git --version` |

**Instalación rápida si no los tienes:**

```bash
# Node.js — descargar desde https://nodejs.org (LTS recomendado)
# Verificar después de instalar:
node --version    # Debe ser >= 18.x
npm --version     # Debe ser >= 9.x

# AWS CLI — descargar desde https://aws.amazon.com/cli/
# Verificar:
aws --version

# Configurar credenciales del sandbox:
aws configure
# → AWS Access Key ID: (tu key del sandbox)
# → AWS Secret Access Key: (tu secret del sandbox)
# → Default region: us-east-1
# → Default output format: json

# Verificar que las credenciales funcionan:
aws sts get-caller-identity
```

**Paquetes npm del proyecto** (se instalan automáticamente con el prompt al agente):

```bash
# Dependencias de producción:
@langchain/core          # Core de LangChain
@langchain/aws           # Integración con Bedrock (Claude + Titan)
@langchain/community     # Loaders de documentos
langchain                # Framework RAG

```

> **Nota:** No necesitas instalar estas librerías manualmente. El agente las instalará cuando ejecute el prompt. Solo asegúrate de tener Node.js, npm y AWS CLI listos.

---

### 2.2 — Validar conexión a AWS y Bedrock

Antes de continuar, valida que tienes acceso correcto a AWS y a los modelos de Bedrock:

**1. Validar conexión a AWS:**
```bash
aws sts get-caller-identity
```
→ Debe retornar tu Account, UserId y Arn. Si da error, revisa `aws configure`.

**2. Validar acceso a Bedrock (listar modelos Claude disponibles):**
```bash
aws bedrock list-foundation-models --region us-east-1 --query "modelSummaries[?contains(modelId, 'anthropic')].[modelId]" --output table
```
→ Debe mostrar modelos de Claude (anthropic.claude-3-sonnet, etc.). Si da error de permisos, Bedrock no está habilitado en la cuenta.

**3. Validar acceso a Titan Embeddings:**
```bash
aws bedrock list-foundation-models --region us-east-1 --query "modelSummaries[?contains(modelId, 'titan-embed')].[modelId]" --output table
```
→ Debe mostrar Amazon Titan Embed. Si no aparece, solicitar acceso en AWS Console → Bedrock → Model access.

**4. Probar invocación directa a Claude Sonnet (el modelo que usaremos):**
```bash
aws bedrock-runtime invoke-model --region us-east-1 --model-id anthropic.claude-3-sonnet-20240229-v1:0 --content-type application/json --accept application/json --body "{\"anthropic_version\":\"bedrock-2023-05-31\",\"max_tokens\":100,\"messages\":[{\"role\":\"user\",\"content\":\"Hola, responde con un saludo breve\"}]}" response.json

type response.json
```
→ Debe retornar una respuesta JSON con el saludo de Claude. Si funciona, tienes todo listo para el RAG.

**Si algo falla:**
- `sts get-caller-identity` falla → ejecutar `aws configure` con las credenciales correctas del sandbox
- Bedrock no lista modelos → ir a AWS Console → Bedrock → Model access → Request access a Claude Sonnet y Titan Embed
- "AccessDeniedException" → la cuenta del sandbox necesita permisos de Bedrock, contactar al administrador

> ✅ Si los 4 pasos pasan, estás listo para el ejercicio.

### 2.3 — Prompt al agente

Abre el chat de Kiro (modo Vibe) y pega este prompt:

```
Necesito que crees un proyecto RAG (Retrieval Augmented Generation) con las siguientes características:

- Stack: Node.js + TypeScript
- Framework: LangChain (@langchain/core, @langchain/aws, langchain)
- LLM: Amazon Bedrock con Claude Sonnet (ChatBedrockConverse)
- Embeddings: Amazon Bedrock Titan Embed (BedrockEmbeddings)
- Vector Store: MemoryVectorStore (en memoria, sin infra externa)
- Documentos: archivos .md en una carpeta docs/

El RAG debe:
1. Cargar todos los .md de la carpeta docs/
2. Dividirlos en chunks con RecursiveCharacterTextSplitter
3. Generar embeddings con Titan y guardarlos en MemoryVectorStore
4. Exponer una función queryRAG(question: string) que:
   - Busque chunks relevantes
   - Los pase como contexto a Claude
   - Retorne la respuesta con las fuentes citadas (nombre del archivo + contenido del chunk)
5. Incluir un script main.ts interactivo que al ejecutarse:
   - Muestre un mensaje de bienvenida con la cantidad de documentos indexados
   - Pida al usuario que ingrese una pregunta por terminal (input interactivo con readline)
   - Procese la consulta con el RAG
   - Muestre la respuesta con las fuentes citadas (nombre del archivo + fragmento)
   - Pregunte si quiere hacer otra consulta o salir
   - Loop interactivo hasta que el usuario escriba "salir" o "exit"

Configura el proyecto completo: package.json, tsconfig.json, y estructura de carpetas.
Región AWS: us-east-1
```

**Deja que el agente trabaje.** Observa cómo crea archivos, instala dependencias y estructura el proyecto.

## Paso 4: Verificar credenciales AWS

En la terminal integrada de Kiro:

```bash
aws sts get-caller-identity
```

Debe retornar tu cuenta del sandbox. Si no, configura:

```bash
export AWS_ACCESS_KEY_ID=tu_key
export AWS_SECRET_ACCESS_KEY=tu_secret
export AWS_DEFAULT_REGION=us-east-1
```
