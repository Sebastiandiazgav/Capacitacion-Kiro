# 1.2 Plataforma Agéntica — Ejercicio Práctico

## Objetivo

Crear un agente custom en Kiro que analice código Python y genere un reporte de calidad, demostrando cómo funciona la plataforma agéntica en la práctica.

---

## Demostración: Agente que analiza código

### Paso 1: Crear la carpeta de agentes

En la raíz de tu proyecto, crea la siguiente estructura:

```
.kiro/
└── agents/
    └── code-analyzer.md
```

### Paso 2: Configurar el agente

Crea el archivo `.kiro/agents/code-analyzer.md` con el siguiente contenido:

```markdown
---
name: Code Analyzer
description: Analiza código Python y genera un reporte de calidad
---

# Code Analyzer Agent

## Rol
Eres un agente especializado en análisis de calidad de código Python.

## Instrucciones
1. Lee el archivo que se te indique
2. Analiza los siguientes aspectos:
   - Nombres de variables y funciones (¿son descriptivos?)
   - Manejo de errores (¿hay try/except donde corresponde?)
   - Imports no utilizados
   - Uso de palabras reservadas como nombres de parámetros
   - Validación de inputs
   - Vulnerabilidades de seguridad (SQL injection, etc.)
   - Convenciones de Python (PEP 8)
   - Complejidad ciclomática
   - Código duplicado
3. Genera un reporte estructurado con:
   - Resumen ejecutivo
   - Problemas encontrados (severidad: alta/media/baja)
   - Recomendaciones específicas con ejemplo de código corregido

## Formato de salida
Genera el reporte en Markdown.
```

### Paso 3: Ejecutar el agente (en Kiro Vibe)

Abre el chat de Kiro (modo Vibe) y escribe:

```
Analiza el archivo 1.2code.py usando las instrucciones del agente code-analyzer. 
Genera un reporte completo de calidad de código.
```

### Paso 4: Revisar el reporte

El agente generará un reporte identificando problemas como:
- Import `random` no utilizado
- Parámetro `type` usa palabra reservada de Python
- SQL injection en `get_users()`
- Función `process()` con nombre no descriptivo
- Clase `userManager` no sigue PascalCase
- Código sin protección `if __name__ == "__main__"`
- Y más...

---

## Ejecución bajo demanda (Kiro CLI)

El agente se puede ejecutar en diferentes entornos:

### Terminal local
```bash
kiro agent run code-analyzer --path ./1.2code.py
```

### Step de GitHub Actions
```yaml
- name: Análisis de código
  run: kiro agent run code-analyzer --path ./src --output report.md
```

### Cron en servidor
```bash
# Crontab: ejecutar análisis diario a las 6am
0 6 * * * cd /path/to/project && kiro agent run code-analyzer --output /reports/daily.md
```

## Ejercicio para el equipo

### Tu turno: Analiza `1.2codeprueba.py`

1. **Revisa** el archivo `1.2codeprueba.py` (no lo modifiques aún)
2. **Crea** tu propio agente en `.kiro/agents/` (puedes usar el mismo `code-analyzer.md` o crear uno con tu toque personal)
3. **Ejecuta** el análisis sobre `1.2codeprueba.py` usando el chat Vibe:
   ```
   Analiza el archivo 1.2codeprueba.py siguiendo las instrucciones del agente code-analyzer.
   Genera un reporte completo.
   ```
4. **Compara** tu reporte con el de tus compañeros
5. **Bonus**: Pide al agente que corrija los problemas encontrados y genere una versión mejorada del código

### Problemas que deberías encontrar en `1.2codeprueba.py`:
- Imports no utilizados (`subprocess`)
- Constantes mal nombradas (`min` en minúscula)
- Uso de built-ins como nombres (`format`, `id`)
- Clase sin PascalCase (`order_processor`)
- Lógica de búsqueda duplicada
- Archivos abiertos sin `with`
- Comparaciones con `== None` y `== True`
- Números mágicos en descuentos
- Sin `if __name__ == "__main__"`

## Concepto clave

> "Creas un agente como configuración en `.kiro/agents/` y lo ejecutas bajo demanda donde quieras — terminal local, step de pipeline, o servidor con cron."

El agente no necesita APIs externas. Trabaja con el contexto de tu proyecto y las herramientas que Kiro ya tiene integradas (lectura de archivos, análisis de código, generación de reportes).
