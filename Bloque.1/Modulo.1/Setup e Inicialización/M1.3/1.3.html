01 — Concepto

¿Qué es RAG?

RAG (Retrieval Augmented Generation) es un patrón donde el modelo de IA no responde solo de memoria — primero busca información relevante en tus documentos y luego genera una respuesta fundamentada con fuentes citadas. Es la forma más directa de hacer que un LLM "conozca" la documentación interna de tu equipo.

Flujo de un RAG

Pregunta → Búsqueda en documentos → Respuesta con contexto

¿Por qué RAG y no fine-tuning?

RAG no modifica el modelo — solo le da contexto en cada consulta. Es más rápido de implementar, más barato de mantener y actualizable en tiempo real (cambias un doc y listo).

Caso AFP UNO

Normativas de la Superintendencia, procedimientos de cotizaciones, guías de deploy, FAQ operativa. El equipo pregunta en lenguaje natural y recibe respuestas con la fuente exacta.

02 — Stack técnico

LangChain + Bedrock + Vector Store en memoria

Todo corre local. No necesitas Knowledge Base preconfigurado, ni S3, ni bases de datos externas. Los documentos se cargan desde archivos locales, se procesan en memoria y se consultan contra Bedrock.

📦

Node.js + TypeScript

Runtime y lenguaje base. Inicializamos con npm init y configuración TypeScript estándar.

🔗

LangChain

@langchain/core, @langchain/aws, langchain — framework que abstrae el flujo RAG completo en componentes reutilizables.

🧠

Claude via Bedrock

Modelo de lenguaje que genera las respuestas. Se invoca vía ChatBedrockConverse de LangChain.

🔢

Titan Embeddings

Modelo de Amazon para convertir texto en vectores numéricos. Vía BedrockEmbeddings.

🗄️

MemoryVectorStore

Vector store en memoria de LangChain. Cero config, cero infra. Ideal para workshops y prototipos rápidos.

📄

Docs locales (.md)

Archivos Markdown que simulan documentación interna de AFP UNO. Se cargan con DirectoryLoader.

Dependencias del proyecto

npm install @langchain/core @langchain/aws @langchain/community langchain npm install @aws-sdk/client-bedrock-runtime npm install typescript tsx @types/node --save-dev

04 — Workshop paso a paso

Construye tu RAG en 30 minutos

Sigue estos pasos usando Kiro en modo Vibe. El agente hace el trabajo pesado — tú guías con prompts y verificas resultados.

Crear workspace

Abre Kiro → File → Open Folder → crea una carpeta nueva para el proyecto RAG. Abre el chat Vibe.

Pedir scaffold al agente

En el chat, escribe el prompt de scaffold (ver archivo 1.3 rag-basico-vibe.md para el prompt exacto). El agente generará la estructura completa.

Agregar documentación interna

Copia los archivos de 1.3docs/ a la carpeta docs/ de tu proyecto. Son documentos simulados de AFP UNO.

Verificar credenciales AWS

Confirma que tus credenciales del sandbox están configuradas: aws sts get-caller-identity en terminal.

Ejecutar el RAG

El agente ejecutará el proyecto. Verás la indexación de documentos y podrás hacer tu primera consulta.

Probar consultas

"¿Cuál es el procedimiento para cotizaciones?" — El RAG debe responder con fuentes citadas del documento interno.

Explorar Timeline

Revisa el timeline de Kiro: ¿cuántos archivos creó? ¿qué comandos ejecutó? ¿usó fs_write y execute_pwsh?
